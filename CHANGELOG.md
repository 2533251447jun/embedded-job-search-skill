# Changelog

All notable changes to `embedded-job-search-skill` are documented here.

This file follows the spirit of [Keep a Changelog](https://keepachangelog.com/) while staying concise for a standalone Skill project.

## [0.1.1] - 2026-06-30

### Added

- Recorded the current project state as `embedded-job-search-skill v0.1.1`.
- Documented the v0.1.1 targeted enhancement focused on practical embedded job-search use, not a project rewrite.
- Captured expanded support for embedded resume project evidence, JD analysis, hardware debug records, application messages, and role-specific examples.

### Changed

- Strengthened reusable templates for project bullets, hardware debugging, JD analysis, and application messages.
- Expanded embedded examples for junior candidates, including STM32/MCU, embedded testing, industrial control, RS485/Modbus, and hardware joint-debug scenarios.
- Improved reference coverage for role keywords, technical maps, Chinese/English keyword mapping, common protocols, and debugging tools.

### Public-Ready Cleanup

- Replaced the local maintainer copyright placeholder with the public GitHub username `2533251447jun`.
- Removed local Windows absolute paths from `README.md` before public repository upload.
- Updated the acceptance checklist so public release checks no longer treat placeholder copyright text as valid.

### Preserved

- Kept the existing `Skill Router + references/templates/examples/tests` architecture.
- Kept heavy embedded knowledge in `references/` instead of turning `SKILL.md` into a technical encyclopedia.
- Kept truthfulness boundaries around `fact`, `inference`, `needs_proof`, `not_allowed`, and `evidence_status`.
- Kept platform boundaries: the Skill may draft messages and checklists, but must not auto-apply, auto-DM, auto-send, auto-contact HR, or bypass platform rules.
- Kept the junior-candidate boundary: course, training, personal, and demo projects must not be rewritten as company, customer, production, or mass-production work.

### Not Changed

- Did not rewrite the project structure.
- Did not change `SKILL.md` frontmatter.
- Did not change `LICENSE`.
- Did not change `NOTICE.md`.
- Did not change the `career-ops` TRADEMARK boundary, source-project non-official status, or no-special-permission statement.
- Did not present this project as an official version of `career-ops` or `JobOK`.
